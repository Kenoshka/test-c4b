**ТЕСТОВОЕ ЗАДАНИЕ**

Вайбкодер

**О позиции**

**Вайбкодер** — специалист, который решает технические задачи с помощью AI-инструментов (Claude, ChatGPT, Cursor и др.), не обладая глубоким бэкграундом в классическом программировании. Главное — умение правильно ставить задачи AI, разбираться в документации и доводить результат до рабочего состояния.

**Задание: Развернуть OpenClaw на своём компьютере**

**OpenClaw** — open-source персональный AI-ассистент, который работает как шлюз (Gateway) и подключается к мессенджерам (Telegram, WhatsApp, Discord и другие).

Репозиторий: [github.com/openclaw/openclaw](https://github.com/openclaw/openclaw)

Документация: [docs.openclaw.ai](https://docs.openclaw.ai)

**Что нужно сделать**

**Развернуть рабочий экземпляр OpenClaw на своём компьютере и подключить к нему Telegram-бота**, который будет отвечать на сообщения как AI-ассистент.

**Этап 1 — Установка**

* Установить Node.js 22.16+ (рекомендуется Node 24).

* Установить OpenClaw глобально через npm.

* Пройти онбординг (openclaw onboard).

* Работает на macOS, Linux и Windows (через WSL2).

**Этап 2 — Подключение AI-модели**

* Получить API-ключ от одного из провайдеров (OpenAI, Anthropic, Google — на ваш выбор, подойдёт бесплатный тир).

* Настроить модель в конфигурации OpenClaw (\~/.openclaw/openclaw.json).

* Убедиться, что ассистент отвечает через CLI (openclaw agent \--message "Привет").

**Этап 3 — Telegram-бот**

* Создать бота через @BotFather в Telegram.

* Подключить бота к OpenClaw (настроить channels.telegram в конфигурации).

* Проверить, что бот отвечает на сообщения в личке.

**Этап 4 — Скринкаст**

* Записать короткое видео (Loom, OBS или запись экрана на телефоне) на 2–5 минут, где вы:

  * Показываете работающий Gateway (терминал или веб\-интерфейс).

  * Отправляете сообщение боту в Telegram и получаете ответ.

  * Кратко рассказываете, какие шаги выполнили и какие трудности встретили.

**Критерии оценки**

| Критерий | Вес | Что смотрим |
| :---- | :---: | :---- |
| **Результат** | 40% | Бот в Telegram отвечает на сообщения. Gateway запущен и работает. |
| **Самостоятельность** | 25% | Кандидат сам разобрался в документации и решил проблемы по ходу. |
| **Использование AI** | 20% | Кандидат использовал AI-инструменты для решения задачи (можно показать скриншоты чатов). |
| **Документирование** | 15% | Скринкаст записан, шаги понятны, трудности описаны честно. |

**Бонусные баллы (необязательно)**

* Написать кастомный SOUL.md — персональную инструкцию для ассистента (тон, стиль, ограничения).

* Поднять через Docker вместо прямой установки.

**Что мы предоставляем**

Эту инструкцию и ссылку на репозиторий. Больше ничего — задача в том, чтобы разобраться самостоятельно.

**Формат сдачи**

1. Ссылка на Google Drive с записью скринкаста.

2. Username Telegram-бота (чтобы мы могли проверить в реальном времени, пока Gateway запущен).

3. Краткое текстовое описание (3–5 предложений): что использовали, что было сложно, сколько времени заняло.

**Дедлайн**
